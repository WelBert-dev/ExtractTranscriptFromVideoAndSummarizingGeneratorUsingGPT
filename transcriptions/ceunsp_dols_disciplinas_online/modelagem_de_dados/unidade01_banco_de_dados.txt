Olá, estudante! Nesta unidade, vamos estudar aspectos fundamentais do Banco de Dados. Vamos entender como ocorreu sua evolução e suas aplicações na atualidade. Vem comigo! O Banco de Dados surgiu para armazenar informações que conhecemos como persistência. E existem bancos de dados específicos para diferentes situações. Nesse contexto, sua evolução tem ocorrido de forma constante, mudando a sua autonomia frente à velocidade e suas formas de utilização. Vamos conhecer melhor essa trajetória? A difusão do Banco de Dados começou por volta de 1960, com o surgimento tipo hierárquico, caracterizado por uma forma de pirâmide ramificada do responsável até os filhotes. Já na década de 1970, ocorreu o grande boom do termo, com a criação do Banco de Dados Relacional pelo matemático Edgar Codde. Elaborado a partir da teoria dos conjuntos, o Banco de Dados Relacional foi bastante difundido e é o mais utilizado ainda hoje para armazenar informações de clientes. Por volta de 1980, foi desenvolvido uma linguagem padrão para os bancos relacionais, chamada de SQL. Ela é definida como uma linguagem estruturada de consulta, e hoje é a norma para acesso aos bancos relacionais, independente do fabricante. Nos anos 1990, ocorreu uma transição do conceito de processamento de dados para a tecnologia da informação. Isso permitiu a inclusão da tomada de decisões no processo, gerando a mineração de dados. Ou seja, foi iniciada a busca de informações que auxiliem na tomada de decisão, também conhecida como Data Mining. Com a proliferação dos serviços para a Web, esses tipos de bancos se tornaram cada vez mais aplicados ao mercado. Dos anos 2000 para frente, a mineração de dados adquiriu uma abrangência maior, sendo aplicada também ao contexto de redes sociais e computação em nuvem, por exemplo. Assim, os bancos de dados podem ser considerados sensores que captam informações do dia a dia das pessoas. Por exemplo, se você curte ou interage com determinadas páginas no Facebook ou no Instagram, o banco capta para poder analisar e adequar serviços ao seu perfil. Portanto, houve uma mudança de foco. O objetivo não é apenas relacionado à persistência, mas também é distinguir perfis e ofertar produtos. Nesse contexto, temos o SGBD, que é o sistema gerenciador de banco de dados. Ele é um software que gerencia aspectos relativos a usuários, perfis e controle de acesso, ou seja, o SGBD cuida de toda parte de segurança e armazenamento do banco. Dentro dele, eu vou ter vários pacotes e funções que permitem que o administrador de banco de dados, ou DBA, possa identificar, rodar queries problemáticas e dar uma performance melhor ao sistema. Nesse cenário, temos o TUNING, que é uma ação que visa a otimização do desempenho do banco de dados. Prevenindo a formação de um gargalo. Por exemplo, temos a execução de uma aplicação que está demorando para responder, como o banco de dados levando mais tempo para extrair a informação. Então, existem essas propriedades do SGBD que permitem o gerenciamento para que o banco de dados possa administrar essa ação. Vamos falar sobre os diferentes tipos de bancos de dados? Nós temos o modelo hierárquico, que é geralmente utilizado em servidores de grande porte, como bancos. O regidit do Windows é um exemplo que utiliza esse sistema hierárquico, ou seja, é feito em árvores na qual encontramos os filhotes dentro da árvore pai. Também podemos definir como diretórios ou pastas, e eles obedecem sempre quem é o pai. Então, temos a parte superior e os que estão abaixo dele, e as informações são encontradas dentro daquela respectiva pasta. Esse modelo é bastante utilizado principalmente por sistemas de mainframes. Outro modelo é o sistema de rede, que geralmente resulta em uma estrutura de rede complexa. Um exemplo desse sistema é um banco de dados relacionado aos amigos do Facebook e de outras redes sociais similares. No Facebook você tem uma pessoa e seus amigos em comum, que estão entrelaçados a partir de um banco de dados de rede. Em comparação ao modelo hierárquico, o registro filho pode possuir diversos pais no sistema de rede. Isso porque os nós podem ter tantas conexões possíveis. Ou seja, eu tenho um amigo nesse grupo do Facebook que pode ter vários outros amigos que se conhecem entre eles. Já o modelo relacional tem a finalidade de guardar informações, ou a persistência. Então, quando utilizamos um SGBD voltado ao banco de dados relacional, buscamos armazenar informações. Por exemplo, vamos ter um cadastro de alunos, de contas, de endereços, de clientes, de pedidos, entre outros. Como vimos, o Edgar Codd se baseou na teoria dos conjuntos para desenvolver o SGBD. Então, ele utilizou as intersecções e cada conjunto. Vamos observar o exemplo das nossas tabelas. Então, eu tenho o conjunto A como o cadastro de fornecedor, enquanto o conjunto B representa o cadastro de cliente e o conjunto C, o de fornecedores. Eles têm a intersecção C de pessoa, ou seja, esses conjuntos estão relacionados por meio do cadastro de pessoas. Esse é um conceito que prova que a matemática está aplicada a vários contextos. Principalmente na nossa área de TI. Temos ainda o banco de dados orientado a objetos. Um modelo bastante aplicado desde os anos 2000 e 2010 por causa da mudança do conceito das linguagens, que passaram a ser orientadas a objetos. E qual é a diferença desse tipo de banco de dados? Ele aplica tudo o que o relacional faz, mas traz também outros conceitos da teoria da orientação a objeto. Isso quer dizer que eu consigo fazer generalização, herança e poliformismo, entre outras aplicações. Por exemplo, eu crio um cadastro de clientes que pode ser a tabela PAI, e depois eu tenho o cadastro de clientes, pessoa física e pessoa jurídica. Ambos são clientes, mas têm especificidades diferentes. Então, eu posso herdar da tabela PAI, algo que não acontece no modelo relacional. Como exemplos de sistemas de gerenciamento de banco de dados orientados a objetos, podemos citar o caché e o jasmine. As informações são úteis para relatar os resultados e as operações atuais de uma organização, colaborando com a gestão de estratégias do negócio. Nesse contexto, um banco de dados é muito importante, porque gerencia e possibilita o acesso às informações. Antigamente existiam sistemas que eram simplesmente informativos, ou páginas estáticas. Hoje, é muito raro fazermos algum sistema que não tenha interação com o banco de dados, pois o processo se tornou muito mais dinâmico, exigindo a mudança de foco de acordo com o perfil da pessoa ou cliente. Ou seja, o sistema precisa ter uma atuação mais inteligente, que permita o armazenamento e a consulta para a tomada de decisão. Por isso, são utilizados os bancos de dados. Assim, estudamos aspectos fundamentais do banco de dados. Abordamos seu surgimento e evolução além dos diferentes modelos e importância de sua utilização. Bons estudos e até a próxima!
